| Volume Type                                     | Where the data is stored                                      | Persists after Pod deletion?   | Explanation                                                                      |
| ----------------------------------------------- | ------------------------------------------------------------- | ------------------------------ | -------------------------------------------------------------------------------- |
| **emptyDir**                                    | Node’s local disk (usually under `/var/lib/kubelet/pods/...`) | ❌ No                           | Created when Pod starts, deleted when Pod stops. Used for temporary data.        |
| **ConfigMap / Secret**                          | In-memory (tmpfs) file system on the node                     | ❌ No                           | Created in memory by kubelet and mounted inside Pod. Deleted with Pod.           |
| **hostPath**                                    | Directly uses a folder on the Node’s disk                     | ✅ Yes (until manually removed) | Mounts a path from the node into the container. Useful for debugging, logs, etc. |
| **PersistentVolume (PVC)**                      | External storage (Azure Disk, AWS EBS, NFS, etc.)             | ✅ Yes                          | Actual disk in cloud or NFS share. Data stays even after Pod is deleted.         |
| **CSI Volume (Secrets Store, Key Vault, etc.)** | Depends on driver (memory, file share, cloud disk)            | Usually ❌                      | Injected dynamically by CSI plugin (for secrets, credentials, etc.)              |
